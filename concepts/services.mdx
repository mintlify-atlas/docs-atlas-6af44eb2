## What Are Services?

Services are **business logic features** that deliver value to your users. Unlike [components](/concepts/components) which focus on infrastructure, services implement domain-specific functionality like authentication, AI interactions, and communications.

Services are defined in `aegis/core/services.py:46-95` and build on top of infrastructure components.

<Note>
Services focus on **what** your application does (business logic), while components focus on **how** it runs (infrastructure).
</Note>

## Service Types

Services are classified by their business purpose:

```python
class ServiceType(Enum):
    AUTH = "auth"                    # Authentication and authorization
    PAYMENT = "payment"              # Payment processing
    AI = "ai"                        # AI and ML integrations
    NOTIFICATION = "notification"    # Email, SMS, push notifications
    ANALYTICS = "analytics"          # Usage analytics and metrics
    STORAGE = "storage"              # File storage and CDN
```

## Available Services

### Authentication (Auth)

<Card title="Auth Service" icon="lock">
  **Type**: `ServiceType.AUTH`
  
  **Description**: User authentication and authorization with JWT tokens
  
  **Use When**: You need user accounts, login/logout, and protected endpoints
  
  **Required Components**:
  - `backend` - For API endpoints
  - `database` - For user storage
  
  **Python Packages**:
  - `python-jose[cryptography]==3.3.0` - JWT handling
  - `passlib[bcrypt]==1.7.4` - Password hashing
  - `python-multipart==0.0.9` - Form data parsing
  
  **Template Files**:
  - `app/components/backend/api/auth/` - Auth endpoints
  - `app/models/user.py` - User model
  - `app/services/auth/` - Auth service logic
  - `app/core/security.py` - Security utilities
</Card>

<Accordion title="Auth Service Implementation">
The auth service provides complete JWT-based authentication:

**User Model** (`app/models/user.py`):
```python
from sqlmodel import SQLModel, Field

class User(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True)
    email: str = Field(unique=True, index=True)
    hashed_password: str
    is_active: bool = True
```

**API Endpoints** (`app/components/backend/api/auth/`):
- `POST /auth/register` - User registration
- `POST /auth/login` - JWT token generation
- `GET /auth/me` - Get current user

**Usage in routes**:
```python
from app.core.security import get_current_user
from fastapi import Depends

@router.get("/protected")
async def protected_route(user: User = Depends(get_current_user)):
    return {"user": user.email}
```

Enable in your project:
```bash
aegis init my-app --services auth
# Or add later:
aegis add database  # Required dependency
# Then enable auth via aegis init with auth service
```
</Accordion>

<Info>
The auth service automatically creates database migrations for the user table. Run `make migrate` after enabling auth.
</Info>

<Accordion title="AI Framework Options">

  choices:

**PydanticAI** (default):

## Service Dependencies

Services declare required components and other services:

### Required Components

From `aegis/core/services.py:24-44`:

```python
@dataclass
class ServiceSpec:
    name: str
    type: ServiceType
    description: str
    required_components: list[str]  # Components needed by this service
    # ...
```

Example from auth service (`aegis/core/services.py:48-64`):

```python
"auth": ServiceSpec(
    name="auth",
    type=ServiceType.AUTH,
    description="User authentication and authorization with JWT tokens",
    required_components=["backend", "database"],  # Must have both
    # ...
)
```

### Service Validation

The system validates service dependencies during initialization (`aegis/core/services.py:132-170`):

```python
def validate_service_dependencies(
    selected_services: list[str],
    available_components: list[str]
) -> list[str]:
    """Validate that all required components are available."""
    errors = []
    
    for service_name in selected_services:
        service = SERVICES[service_name]
        
        # Check required components
        for required_comp in service.required_components:
            if required_comp not in available_components:
                errors.append(
                    f"Service '{service_name}' requires component '{required_comp}'"
                )
    
    return errors
```

## Services vs Components

Understanding the distinction:

| Aspect | Components | Services |
|--------|-----------|----------|
| **Focus** | Infrastructure | Business Logic |
| **Purpose** | Technical capabilities | User-facing features |
| **Examples** | Database, Workers, Redis | Auth, AI, Communications |
| **Dependencies** | Other components | Components + other services |
| **Lifecycle** | Can be added/removed | Configured at init time |
| **Location** | `app/components/` | `app/services/` |

<Info>
Components provide the **foundation**, services deliver the **value**.
</Info>

## Service Configuration

Services are configured during project initialization:

```bash
# Single service
aegis init my-app --services auth

# Multiple services
aegis init my-app --services auth,ai,comms

# Interactive selection
aegis init my-app
# Then select services during prompts
```

Configuration is stored in `copier.yml:156-224`:

```yaml
# Service selection
include_auth: bool
include_ai: bool
include_comms: bool

# Service-specific configuration
ai_framework: "pydantic-ai" | "langchain"
ai_providers: str  # Comma-separated
ai_backend: "memory" | "sqlite" | "postgres"
ai_rag: bool
ai_voice: bool
```

## Service Registry

All services are registered in `SERVICES` dict (`aegis/core/services.py:46-95`):

```python
SERVICES: dict[str, ServiceSpec] = {
    "auth": ServiceSpec(...),
    "ai": ServiceSpec(...),
    "comms": ServiceSpec(...),
}
```

Access service information:

```python
from aegis.core.services import get_service, list_available_services

# Get specific service
auth = get_service("auth")
print(auth.description)  # "User authentication..."
print(auth.required_components)  # ['backend', 'database']

# List all services
all_services = list_available_services()
print(all_services)  # ['auth', 'ai', 'comms']
```

## Adding Services After Init

<Warning>
Currently, services must be selected during `aegis init`. Unlike components, there is no `aegis add service` command yet.
</Warning>

To add a service to an existing project:

<Steps>
  <Step title="Ensure required components are present">
    ```bash
    # For auth: need database
    aegis add database
    ```
  </Step>
  <Step title="Run aegis update to regenerate project">
    ```bash
    # This will prompt for service selection
    aegis update
    ```
  </Step>
  <Step title="Manually edit .copier-answers.yml">
    ```yaml
    # Add service configuration
    include_auth: true
    ```
  </Step>
  <Step title="Re-run copier update">
    ```bash
    copier update
    ```
  </Step>
</Steps>

<Info>
Service lifecycle management (add/remove commands) is planned for a future release.
</Info>

## Service Specification Reference

Full `ServiceSpec` definition from `aegis/core/services.py:24-44`:

```python
@dataclass
class ServiceSpec:
    """Specification for a single service."""
    
    name: str                                      # Service identifier
    type: ServiceType                              # Business classification
    description: str                               # Human-readable description
    required_components: list[str] = []           # Components this needs
    recommended_components: list[str] = []        # Soft dependencies
    required_services: list[str] = []             # Other services needed
    conflicts: list[str] = []                     # Incompatible services
    pyproject_deps: list[str] = []                # Python packages
    template_files: list[str] = []                # Template paths to include
```
