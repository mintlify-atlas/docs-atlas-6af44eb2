**What if you could go from idea to working prototype in the time it takes to grab coffee?**

## Quick Start

Get up and running in minutes:

<CodeGroup>

```bash uvx (instant)
uvx aegis-stack init my-api
cd my-api && uv sync && cp .env.example .env && make serve
```

```bash pip
pip install aegis-stack
aegis init my-api
cd my-api && uv sync && cp .env.example .env && make serve
```

```bash uv tool
uv tool install aegis-stack
aegis init my-api
cd my-api && uv sync && cp .env.example .env && make serve
```

</CodeGroup>

Your API will be running at `http://localhost:8000` with interactive docs at `http://localhost:8000/docs`.

## Stack Evolution

**Your choices aren't permanent.** Start with what you need today, add components when requirements change, remove what you outgrow.

```bash
# Monday: Ship MVP
aegis init my-api

# Week 3: Add scheduled reports
aegis add scheduler --project-path ./my-api

# Month 2: Need async workers
aegis add worker --project-path ./my-api

# Month 6: Scheduler not needed
aegis remove scheduler --project-path ./my-api

# Stay current with template improvements
aegis update
```

<Note>
Most starters lock you in at `init`. Aegis Stack doesn't. Learn more in [Evolving Your Stack](/concepts/evolution).
</Note>

## Available Components & Services

### Components (Infrastructure)

- **Backend** → FastAPI + Pydantic V2 + Uvicorn
- **Frontend** → Flet UI framework
- **Database** → Postgres / SQLite with SQLModel
- **Cache/Queue** → Redis
- **Scheduler** → APScheduler for cron-like tasks
- **Worker** → Arq / Taskiq for background jobs
- **Ingress** → Traefik reverse proxy

### Services (Business Logic)

- **Auth** → JWT authentication
- **AI** → PydanticAI / LangChain integration
- **Comms** → Resend + Twilio for email/SMS
