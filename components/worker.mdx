The Worker component provides Redis-based background task processing using Arq. Offload heavy computations, send emails, process files, or handle any long-running operations without blocking your API.

## What It Does

The Worker component:

- **Async Task Processing**: Handle background jobs asynchronously
- **Priority Queues**: Route tasks to high, medium, or low priority queues
- **Functional Queues**: Organize tasks by purpose (system, load_test, media)
- **Redis-Based**: Reliable message queue with persistence
- **Horizontal Scaling**: Run multiple worker instances
- **Task Monitoring**: Track job status and results
- **Retry Logic**: Automatic retries on failure

## When To Use

Add the Worker component when you need:

- **Heavy Processing**: CPU-intensive tasks that block API responses
- **Email/SMS Sending**: Async communication without API delays
- **File Processing**: Upload handling, image resizing, video transcoding
- **External API Calls**: Long HTTP requests to third-party services
- **Batch Operations**: Process large datasets in the background
- **Scheduled Tasks**: Combined with Scheduler for deferred execution

<Warning>
  Workers require the Redis component. It will be automatically added if you include Worker.
</Warning>

## Configuration

Configure workers in your `.env` file:

```bash
# Redis Configuration (required for workers)
REDIS_URL=redis://localhost:6379/0
REDIS_CONN_TIMEOUT=5
REDIS_CONN_RETRIES=3
REDIS_CONN_RETRY_DELAY=1

# Worker Configuration
WORKER_DEFAULT_QUEUE=system          # Default queue for tasks
WORKER_LOAD_TEST_QUEUE=load_test    # Queue for load testing

# Worker Pool Sizes (per priority level)
WORKER_SYSTEM_HIGH_CONCURRENCY=4
WORKER_SYSTEM_MEDIUM_CONCURRENCY=2
WORKER_SYSTEM_LOW_CONCURRENCY=1
```

### Environment Variables

<ResponseField name="REDIS_URL" type="string" default="redis://localhost:6379/0">
  Redis connection URL for worker queues
</ResponseField>

<ResponseField name="WORKER_DEFAULT_QUEUE" type="string" default="system">
  Default functional queue for tasks:
  - `system`: System maintenance tasks
  - `load_test`: Load testing tasks
  - `media`: Media processing tasks (if added)
</ResponseField>

<ResponseField name="WORKER_*_CONCURRENCY" type="integer" default="varies">
  Number of concurrent workers per queue/priority:
  - `WORKER_SYSTEM_HIGH_CONCURRENCY`: High priority (default: 4)
  - `WORKER_SYSTEM_MEDIUM_CONCURRENCY`: Medium priority (default: 2)
  - `WORKER_SYSTEM_LOW_CONCURRENCY`: Low priority (default: 1)
</ResponseField>

```python app/components/worker/pools.py
from arq import create_pool
from arq.connections import ArqRedis

_pool_cache: dict[str, ArqRedis] = {}

async def get_queue_pool(queue_type: str | None = None) -> tuple[ArqRedis, str]:
    """Get Redis pool for enqueueing tasks."""
    queue_type = queue_type or get_default_queue()
    
    # Reuse cached pool
    if queue_type in _pool_cache:
        return _pool_cache[queue_type], get_queue_name(queue_type)
    
    # Create new pool
    pool = await create_pool(RedisSettings.from_dsn(settings.REDIS_URL))
    _pool_cache[queue_type] = pool
    
    return pool, get_queue_name(queue_type)
```

```python app/components/worker/tasks/my_tasks.py
from typing import Any

    user_name: str,

    """
    Send welcome email to new user.
    
    Args:
        ctx: Arq context with job metadata
        user_email: Email address
        user_name: User's name
        
    Returns:
        Task result

    logger.info(f"Sending welcome email to {user_email}")
```

### Priority Queues

Route tasks to appropriate priority levels:

```python
# High priority - processed first
pool, queue_name = await get_queue_pool("system")  # Uses high priority worker
await pool.enqueue_job("urgent_task", _queue_name=queue_name)

# Low priority - processed when high priority queue is empty
pool, queue_name = await get_queue_pool("load_test")  # Uses low priority worker
await pool.enqueue_job("background_cleanup", _queue_name=queue_name)
```

### Checking Job Status

Monitor task execution:

```python

job = await pool.enqueue_job("my_task", _queue_name=queue_name)
```

## Common Patterns

### Retries

Configure automatic retries:

```python
from arq import Retry

async def flaky_task(ctx):
    try:
        # Attempt operation
        result = await external_api_call()
        return result
    except Exception as e:
        # Retry up to 3 times with exponential backoff
        raise Retry(defer=ctx['job_try'] * 5)  # 5s, 10s, 15s
```

### Task Chaining

Enqueue follow-up tasks:

```python
async def step_one(ctx):
    result = await do_work()
    
    # Enqueue next step
    pool, queue_name = await get_queue_pool()
    await pool.enqueue_job(
        "step_two",
        data=result,
        _queue_name=queue_name,
    )
    
    return result

async def step_two(ctx, data):
    final_result = await process_data(data)
    return final_result
```

### Deferred Execution

Schedule tasks to run later:

```python
from datetime import timedelta

# Run task in 1 hour
pool, queue_name = await get_queue_pool()
await pool.enqueue_job(
    "reminder_task",
    user_id=123,
    _queue_name=queue_name,
    _defer_by=timedelta(hours=1),
)
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Tasks not processing">
    Check worker and Redis are running:
    ```bash
    docker-compose ps worker-system redis
    docker-compose logs worker-system
    ```
  </Accordion>

  <Accordion title="Connection to Redis failed">
    Verify Redis URL:
    ```bash
    redis-cli -u redis://localhost:6379 ping
    ```
    
    Check `REDIS_URL` in `.env` matches your Redis instance.
  </Accordion>

  <Accordion title="Tasks timing out">
    Increase timeout in worker configuration:
    ```python
    WorkerSettings(
        job_timeout=600,  # 10 minutes
        # ...
    )
    ```
  </Accordion>

  <Accordion title="Queue backing up">
    Scale up workers:
    ```bash
    docker-compose up -d --scale worker-system=5
    ```
    
    Or increase concurrency:
    ```bash
    WORKER_SYSTEM_HIGH_CONCURRENCY=10
    ```
  </Accordion>
</AccordionGroup>
