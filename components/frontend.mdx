The Frontend component provides **Overseer** - a built-in web dashboard for monitoring your Aegis Stack application. View system health, component status, activity logs, and architecture diagrams in real-time.

## What It Does

The Frontend component (Overseer):

- **System Dashboard**: Real-time view of all components and services
- **Health Monitoring**: Visual status indicators for every part of your stack
- **Activity Feed**: Live stream of system events and operations
- **Architecture Diagram**: Visual representation of your system topology
- **Component Details**: Drill down into metrics and configuration
- **Dark Mode**: Theme toggle for comfortable viewing
- **Responsive Design**: Works on desktop, tablet, and mobile

## When To Use

The Frontend is a **core component** that's automatically included in every Aegis Stack project. You cannot remove it.

Use it to:
- Monitor system health at a glance
- Debug issues by viewing component status
- Track system activity in real-time
- Understand your application's architecture
- Inspect component-specific metrics

<Note>
  Overseer is designed for operational visibility. For user-facing interfaces, build your own frontend or use Flet's capabilities to extend Overseer.
</Note>

## Adding to Your Project

The Frontend component is automatically included:

```bash
aegis init my-api
```

It cannot be added or removed separately since it's a core component.

## Configuration

Configure the frontend through environment variables:

```bash
# Frontend Configuration
FLET_ASSETS_DIR=assets              # Path to static assets
PORT=8000                           # Web server port
```

```python app/components/frontend/main.py
import flet as ft
from app.services.system.models import ComponentStatus

def create_frontend_app() -> Callable:
    """Create the Flet dashboard application."""
    
    async def flet_main(page: ft.Page) -> None:
        page.title = "Aegis Stack - System Dashboard"
        
        # Initialize theme manager
        theme_manager = ThemeManager(page)
        await theme_manager.initialize_themes()
        
        # Create dashboard components
        status_overview = StatusOverviewPanel()
        activity_feed = ActivityFeed()
        diagram_view = DiagramView()
        
        # Component cards
        cards_container = ft.ResponsiveRow(controls=[])
        
        async def refresh_dashboard() -> None:
            """Fetch and update system status."""
            response = await http_client.get("http://localhost:8000/health/")
            data = response.json()
            
            # Update all views
            await update_health_status(data)
            await update_component_cards(data)
            await update_activity_feed()
            page.update()
        
        # Auto-refresh every 30 seconds
        await refresh_dashboard()
        asyncio.create_task(auto_refresh())
    
    return flet_main
```

## Integration with Backend

Overseer fetches data from the backend's health endpoint:

```python
# Backend provides health data
GET /health/

# Response includes all components
{
  "status": "healthy",
  "components": {
    "aegis": {
      "sub_components": {
        "backend": {...},
        "database": {...},
        "worker": {...}
      }
    }
  }
}
```

Components register their health checks:

```python app/services/my_service/health.py
from app.services.system.models import ComponentStatus, ComponentStatusType

async def get_health() -> ComponentStatus:
    """Report component health to Overseer."""
    try:
        # Check your component
        is_healthy = await check_my_component()
        
        return ComponentStatus(
            name="my_component",
            status=ComponentStatusType.HEALTHY if is_healthy else ComponentStatusType.UNHEALTHY,
            message="Component is running" if is_healthy else "Component failed",
            metadata={
                "version": "1.0.0",
                "custom_metric": 42,
            }
        )
    except Exception as e:
        return ComponentStatus(
            name="my_component",
            status=ComponentStatusType.UNHEALTHY,
            message=f"Health check failed: {e}",
        )
```

## Common Patterns

### Real-Time Updates

Overseer auto-refreshes every 30 seconds. To force an immediate refresh:

```python
# Access refresh function from any modal
refresh_fn = page.data.get("refresh_dashboard")
if refresh_fn:
    await refresh_fn()
```

### Component Modals

Show detailed info when cards are clicked:

```python
class MyComponentModal:
    def __init__(self, component_data: ComponentStatus):
        self.component_data = component_data
    
    def show(self, page: ft.Page):
        modal = ft.AlertDialog(
            title=ft.Text("Component Details"),
            content=ft.Column([
                ft.Text(f"Status: {self.component_data.status.value}"),
                ft.Text(f"Message: {self.component_data.message}"),
                # More details...
            ]),
            actions=[ft.TextButton("Close", on_click=lambda _: page.close(modal))],
        )
        page.dialog = modal
        modal.open = True
        page.update()
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Dashboard not loading">
    Check the backend is running:
    ```bash
    curl http://localhost:8000/health/
    ```
    
    Verify port 8000 is accessible.
  </Accordion>

  <Accordion title="Components not showing">
    Ensure components report health via the backend health endpoint. Check backend logs for errors.
  </Accordion>

  <Accordion title="Assets not loading">
    Verify `FLET_ASSETS_DIR` points to correct directory:
    ```
    assets/
    ├── overseer.png
    └── aegis-manifesto.png
    ```
  </Accordion>

  <Accordion title="Slow refresh">
    Reduce refresh interval by modifying `asyncio.sleep(30)` in `auto_refresh()` function.
  </Accordion>
</AccordionGroup>
