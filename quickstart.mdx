## Create Your First Project

Get a working Aegis Stack project up and running in under 5 minutes.

<Steps>
  <Step title="Generate a new project">
    Use `uvx` to instantly create a project without installation:

    ```bash
    uvx aegis-stack init my-api
    ```

    <Accordion title="Want to customize during init?">
      Add components directly at creation:

      ```bash
      # Create with authentication
      uvx aegis-stack init user-app --services auth

      # Create with background processing
      uvx aegis-stack init task-processor --components worker,scheduler

      # Create with database and Redis
      uvx aegis-stack init data-api --components database,redis
      ```
    </Accordion>
  </Step>

  <Step title="Navigate and install dependencies">
    ```bash
    cd my-api
    uv sync
    cp .env.example .env
    ```

    <Info>
    The `.env` file contains configuration for all components. Review and update values as needed for your environment.
    </Info>
  </Step>

  <Step title="Start the development server">
    ```bash
    make serve
    ```

    This command uses Docker Compose to start:
    - FastAPI backend on `http://localhost:8000`
    - Overseer dashboard on `http://localhost:8000` (root URL)
    - Interactive API docs at `http://localhost:8000/docs`
    - Health check endpoint at `http://localhost:8000/health`
  </Step>

  <Step title="Explore your running project">
    <CardGroup cols={2}>
      <Card title="API Documentation" icon="book" href="http://localhost:8000/docs">
        Interactive Swagger UI for testing endpoints
      </Card>
      <Card title="Overseer Dashboard" icon="gauge">
        System visibility dashboard at the root URL
      </Card>
      <Card title="Health Endpoint" icon="heart-pulse">
        `/health` provides component status
      </Card>
      <Card title="ReDoc" icon="file-lines" href="http://localhost:8000/redoc">
        Alternative API documentation view
      </Card>
    </CardGroup>
  </Step>
</Steps>

## Understanding the Generated Structure

Your new Aegis Stack project has a clean, organized structure:

<Tabs>
  <Tab title="Core Structure">
    ```
    my-api/
    ├── app/
    │   ├── components/     # Infrastructure: backend, frontend
    │   ├── services/       # Business logic (when added)
    │   ├── cli/           # Built-in CLI for your app
    │   └── entrypoints/   # Startup scripts
    ├── tests/             # Pytest test suite
    ├── .env.example       # Environment template
    ├── docker-compose.yml # Development services
    ├── Makefile          # Common commands
    └── pyproject.toml    # Dependencies
    ```
  </Tab>

  <Tab title="Components">
    Core components included by default:

    - **Backend** (`app/components/backend/`) - FastAPI server
    - **Frontend** (`app/components/frontend/`) - Overseer dashboard

    Optional components you can add:

    - **Database** - SQLModel with SQLite or PostgreSQL
    - **Worker** - Arq background job processor
    - **Scheduler** - APScheduler for cron jobs
    - **Redis** - Cache and message broker
    - **Ingress** - Traefik reverse proxy
  </Tab>

  <Tab title="Entry Points">
    ```
    app/entrypoints/
    ├── backend.py      # FastAPI server entry point
    ├── frontend.py     # Overseer dashboard entry point
    ├── worker.py       # Worker process (if added)
    └── scheduler.py    # Scheduler process (if added)
    ```

    Each entrypoint is a standalone process that can be run independently or via Docker Compose.
  </Tab>
</Tabs>

## Add Your First Endpoint

Let's add a custom API endpoint to your backend:

<Steps>
  <Step title="Create a new router file">
    Create `app/components/backend/routes/hello.py`:

    ```python app/components/backend/routes/hello.py
    from fastapi import APIRouter

    router = APIRouter(prefix="/hello", tags=["hello"])

    @router.get("/")
    async def hello_world():
        return {"message": "Hello from Aegis Stack!"}

    @router.get("/{name}")
    async def hello_name(name: str):
        return {"message": f"Hello, {name}!"}
    ```
  </Step>

  <Step title="Register the router">
    The backend auto-discovers routers in `app/components/backend/routes/`, so your endpoint is immediately available.

    Test it:

    ```bash
    curl http://localhost:8000/hello
    # {"message": "Hello from Aegis Stack!"}

    curl http://localhost:8000/hello/World
    # {"message": "Hello, World!"}
    ```
  </Step>

  <Step title="View in Swagger UI">
    Visit `http://localhost:8000/docs` to see your new endpoints in the interactive documentation. You can test them directly from the browser.
  </Step>
</Steps>

## Add Infrastructure Components

Need background jobs or a database? Add components on demand:

<CodeGroup>

```bash Worker (background jobs)
aegis add worker
make serve  # Restart to pick up new worker
```

```bash Scheduler (cron jobs)
aegis add scheduler
make serve
```

```bash Database (SQLModel + Postgres)
aegis add database
make serve
```

```bash Redis (cache + queue)
aegis add redis
make serve
```

</CodeGroup>

<Tip>
Components can be added at any time. Read more about [adding components](/guides/adding-components).
</Tip>

## Common Development Commands

Your project includes a `Makefile` with helpful commands:

<CodeGroup>

```bash Development
# Start all services
make serve

# Run tests
make test

# Lint code
make lint

# Format code
make format

# Type check
make typecheck

# Run all checks (lint + type + test)
make check
```

```bash Docker
# View logs
make logs

# Stop all services
make down

# Rebuild containers
make build

# Clean volumes and restart
make clean
```

```bash Database (if database component added)
# Create migration
make migration

# Apply migrations
make migrate

# View migration status
make migration-status
```

</CodeGroup>

## Working with Docker

The generated `docker-compose.yml` manages your development environment:

```yaml docker-compose.yml
services:
  backend:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - .:/app
    environment:
      - ENVIRONMENT=development
```

<Accordion title="Running without Docker">
If you prefer to run components individually:

```bash
# Install dependencies
uv sync

# Run backend directly
uvicorn app.components.backend.app:app --reload --port 8000

# Run frontend (Overseer)
python -m app.entrypoints.frontend

# Run worker (if added)
python -m app.entrypoints.worker

# Run scheduler (if added)
python -m app.entrypoints.scheduler
```

Note: You'll need to manage service dependencies (Redis, PostgreSQL) yourself.
</Accordion>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 8000 already in use">
    Change the port in `docker-compose.yml`:

    ```yaml
    ports:
      - "8001:8000"  # Use port 8001 instead
    ```

    Or stop the conflicting service:

    ```bash
    lsof -ti:8000 | xargs kill -9
    ```
  </Accordion>

  <Accordion title="Docker daemon not running">
    Ensure Docker Desktop is running:

    ```bash
    # macOS/Windows: Start Docker Desktop
    # Linux:
    sudo systemctl start docker
    ```
  </Accordion>

  <Accordion title="Module import errors">
    Ensure dependencies are installed and you're using the correct Python version:

    ```bash
    uv sync
    python --version  # Should be 3.11+
    ```
  </Accordion>

  <Accordion title="Health check fails">
    Check service logs:

    ```bash
    make logs
    ```

    Ensure all required environment variables are set in `.env`.
  </Accordion>
</AccordionGroup>
